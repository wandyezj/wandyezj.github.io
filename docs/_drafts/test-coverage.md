Thinking about building a test plan for a specific project what should you think about?

draft a list of specific areas potentially coverable with tests. Then for each area assess:

- Platform Coverage - Current coverage on each platform.

- Customer Impact - How critical the area is to cover verses other areas? What happens if something goes wrong?

- Regression Likelihood - Overall risk of regressions we think will happen in an area. How likely do we think issues are?

- Fix Time - how long it would take to fix a regression in an area once it occurs.

- Test Flakiness - How likely is the test to produce false positives over time?

- Test Maintenance Cost - How expensive will the test will be to maintain over time?
